# daily-routine

Responsabilidad: aprender horarios y momentos de uso.

Este modulo no programa notificaciones reales. Solo detecta patrones como entrada frecuente por la manana o uso nocturno para que ZENIN pueda saludar o sugerir descanso en una fase posterior.

Relaciones:

- consume observaciones de `observation-engine`;
- alimenta `presence-engine` y `proactive-events`.
