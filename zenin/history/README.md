# history

Responsabilidad: conservar trazabilidad tecnica limitada de eventos, decisiones y cambios internos.

El historial no es memoria emocional ni perfil del usuario. Sirve para auditoria local, depuracion y futuras explicaciones sobre por que ZENIN decidio actuar o no actuar.

Relaciones:

- recibe eventos desde `events`;
- puede alimentar resumenes de `context`;
- debe respetar limites de retencion.
